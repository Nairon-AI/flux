# Pitfalls

Auto-captured from epic review iterations. Organized by area of concern.

```
pitfalls/
  frontend/          # UI, components, layout, forms
  backend/           # Server logic, data processing
  security/          # Auth, permissions, injection, secrets
  async/             # Race conditions, concurrency, event handling
  api/               # Endpoints, validation, contracts
  database/          # Queries, migrations, data integrity
  testing/           # Test gaps, coverage, flaky tests
  infra/             # Deployment, config, monitoring
```

Area directories are created on-the-fly during learning capture. If no existing area fits, create a new one with a short, descriptive slug.

`/flux:meditate` promotes recurring pitfalls into `brain/principles/` and prunes one-offs.
